# Banking-Fraud-Detection-and-Risk-Analytics
This project demonstrates an end-to-end data analytics workflow for detecting banking fraud and analysing transaction risks using Microsoft Excel and Microsoft Power BI. The objective was to transform raw banking transaction data into meaningful insights through data cleaning, visualization, and interactive business intelligence reporting.

The dataset consists of 10,000 banking transactions containing information on transaction amounts, payment channels, authentication methods, device risk scores, account age, login attempts, international transactions, and fraud indicators. Data preprocessing was performed in Microsoft Excel by handling missing values, validating data types, checking for duplicates, renaming columns, and creating derived features such as Transaction Amount Category, Device Risk Category, Account Age Group, and Time of Day.

Exploratory analysis was conducted using Pivot Tables, Pivot Charts, and an interactive Excel dashboard with slicers to identify initial fraud trends. The cleaned dataset was then imported into Microsoft Power BI, where a four-page interactive dashboard was developed using KPI cards, bar charts, donut charts, treemaps, matrix tables, decomposition trees, waterfall charts, funnel charts, and interactive slicers.

The dashboards analyse fraud distribution, payment channels, authentication methods, device risk, transaction timing, account behaviour, and fraud investigation pathways. Key findings highlight patterns in fraudulent transactions across multiple business dimensions, enabling better understanding of fraud risks and supporting data-driven decision-making.

This project showcases practical skills in data cleaning, exploratory data analysis, dashboard development, data visualization, and business intelligence, demonstrating how analytical tools can be used to support fraud detection and risk assessment in the banking industry.
